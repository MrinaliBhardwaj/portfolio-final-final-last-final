## Anti-Patterns Verdict

LLM assessment: NOT AI slop — the opposite failure class. Deeply art-directed, committed, distinctive; the weakness is systems consistency, not genericism.
Deterministic scan: 0 findings across App/Cover/Dock/WorldTabs/FigmaPanel/Gallery/Notes/Pond.
Browser overlays: skipped — hidden preview pane fires no scroll events and never completes framer route transitions (measured), so overlays could not render truthfully.

## Priority Issues

**[P1] Fragmented "go home" system.** Four different affordances (tab ×, chrome close ×, MB link, footer link); monogram is a dead span on the cover, an uppercase MB link in design, mobile-only link in tech, decorative span in gallery/notes/pond. Fix: monogram always top-left, always Pinyon "mb", always links #/; one 44px × top-right in every world; tabs lose their ×.

**[P1] No hierarchy at the cover's settle moment.** Top-nav design/tech links + Explore Design/Tech CTAs + 6-item dock = ~10 choices, 3 visual languages, no primary. Fix: delete the top-nav world links; the split CTAs are the narrative primary; the dock is the system layer.

**[P1] Three concurrent design→tech paths** (WorldTabs, FigmaPanel Pages list, dock). Fix: FigmaPanel Pages becomes display-only; tabs are THE in-fiction sibling switch; dock stays global.

**[P2] Two tab systems for the same two destinations.** Design: fixed full-width Figma bar, labels "design/tech". Tech: sticky editor tabs inside content, labels "design.fig/tech.jsx". Fix: one placement (fixed top bar, skinned per chrome), one label set (design.fig / tech.jsx everywhere).

**[P2] Dead controls in primary chrome.** Cover GitHub is href="#" + preventDefault (real profile exists and is linked in tech world); Claude dock icon renders but does nothing. Fix: real URL; remove Claude until it works.

**[P3] Wipe direction arbitrary for satellite apps; no dock hover name tags; Notes aria-label says "archived first drafts" but the app holds the mini-mri scrapbook.**

## Proposed sitemap

One flat level under the void; hierarchy is dock order, not depth:
`#/` void (launcher) → `#/design`, `#/tech` (flagship worlds, tab-linked siblings) + `#/notes`, `#/gallery`, `#/pond` (satellite apps).

Navigation rules: mb monogram = home (everywhere); × = close app → void (everywhere); tabs = design↔tech only; dock = global app switcher (launch with wipe from cover, crossfade between worlds, scroll-to-top on active); CTAs on cover = narrative entry only.

## Persona Red Flags

**Jordan (first-timer)**: icon-only dock with no visible labels; monogram not clickable on the very first screen (teaches "logo isn't a link"); clicks GitHub on the cover, nothing happens.
**Casey (mobile)**: tabs hidden ≤768px so dock alone switches worlds (good, thumb zone), but before the dock settles the cover has zero nav; top-right × targets are reachable but far.
**Riley (stress tester)**: #/anything silently lands on the cover with no acknowledgment; Claude icon looks interactive, does nothing, no feedback.
